# TodoWrite Workflow

- Use TodoWrite for non-trivial or multi-step work.
- Keep at most one todo item `in_progress` at a time.
- Update todos in parallel with your first exploration calls when practical.
- Do not mark a task completed while tests fail, diagnostics remain, or implementation is partial.
- When the user specifies commands or flags, preserve them exactly in tracked verification steps.
